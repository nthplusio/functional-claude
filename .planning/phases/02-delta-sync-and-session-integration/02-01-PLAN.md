<feature>
  <name>Cache store sync-meta, merge, and freshness functions</name>
  <files>plugins/project-manager/hooks/lib/cache-store.js, plugins/project-manager/hooks/lib/cache-store.test.js</files>
  <behavior>
    readSyncMeta:
    - readSyncMeta(slug, cacheRoot) returns parsed sync-meta.json when file exists and has version field
    - readSyncMeta returns null when file does not exist
    - readSyncMeta returns null when file contains invalid JSON
    - readSyncMeta returns null when parsed data has no version field

    writeSyncMeta:
    - writeSyncMeta(slug, meta, cacheRoot) writes sync-meta.json atomically (temp+rename) and returns true
    - writeSyncMeta returns false (not throws) on permission error
    - No .tmp file remains after successful write
    - Written file contains the exact meta object as JSON

    mergeIssues:
    - mergeIssues(existingData, deltaResult) returns new object with delta issues overlaid on existing
    - Existing issues not in delta are preserved unchanged
    - Issues in both existing and delta use the delta version (update)
    - New issues in delta are added
    - Result has version and tracker from existingData, syncedAt from deltaResult
    - Original existingData object is not mutated

    classifyFreshness:
    - classifyFreshness(syncMeta) returns { tier: 'FRESH', age, message } when lastSyncedAt < 1 hour ago
    - Returns { tier: 'STALE', ... } when lastSyncedAt > 1 hour but lastFullSyncAt within TTL
    - Returns { tier: 'EXPIRED', ... } when lastFullSyncAt > TTL
    - Uses ttlHours from syncMeta (default 24 if missing)
    - Message includes human-readable age via formatAge

    formatAge:
    - formatAge(ms) returns "Xm" for durations under 1 hour
    - Returns "Xh" for durations under 24 hours
    - Returns "Xd" for durations 24+ hours
    - Edge case: 0ms returns "0m"
  </behavior>
  <implementation>
    Add all 5 functions to cache-store.js following existing module patterns (CommonJS, fail-open, cacheRoot override for testability).

    readSyncMeta: Follow readIssues pattern but simpler (no .bak fallback needed -- sync-meta is reconstructable from a full sync). Read file, parse JSON, validate version field, return data or null.

    writeSyncMeta: Follow writeIssues pattern -- getCacheDir for path, writeFileSync to .tmp.<pid>, renameSync to final. No .bak needed (sync-meta is cheap to recreate). Clean up temp on failure. Return boolean.

    mergeIssues(existingData, deltaResult): Shallow copy existingData.issues via spread, overlay deltaResult.issues entries, return new object with { version: existingData.version, tracker: existingData.tracker, syncedAt: deltaResult.syncedAt, issues: merged }.

    classifyFreshness(syncMeta): Compute age from lastSyncedAt and lastFullSyncAt. TTL from syncMeta.ttlHours || 24. Three tiers: EXPIRED if lastFullSyncAt > TTL, STALE if lastSyncedAt > 1 hour, FRESH otherwise. Return { tier, age, message } where message uses formatAge.

    formatAge(ms): Math.floor to minutes. Under 60 = "Xm", under 1440 = "Xh", else "Xd".

    Add all 5 to module.exports alongside existing exports.
  </implementation>
</feature>
