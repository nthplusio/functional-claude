Add delta fetch capabilities to both GitHub and Linear adapters.

Purpose: Delta sync is the core performance optimization -- fetching only changed issues instead of all issues. GitHub uses `gh api` with `since` parameter (server-side filtering). Linear uses client-side timestamp comparison (MCP has no date filter). These adapter extensions are consumed by the pm-status skill (Plan 03) to perform on-demand delta sync.

Output: GitHub adapter with fetchDelta + normalizeGitHubApiIssues; Linear adapter with normalizeLinearDelta.

<feature>
  <name>GitHub and Linear delta adapter methods</name>
  <files>plugins/project-manager/hooks/lib/github-adapter.js, plugins/project-manager/hooks/lib/github-adapter.test.js, plugins/project-manager/hooks/lib/linear-adapter.js, plugins/project-manager/hooks/lib/linear-adapter.test.js</files>
  <behavior>
    normalizeGitHubApiIssues (separate from existing normalizeGitHubIssues):
    - Converts snake_case fields from gh api response (updated_at -> updatedAt in normalized output)
    - Maps lowercase state: "open" -> "started", "closed" -> "completed"
    - Filters out items with pull_request field (PRs returned by GitHub issues endpoint)
    - Extracts priority from labels using existing extractPriorityFromLabels
    - Returns { issues, syncedAt } in same shape as normalizeGitHubIssues
    - Issue keys are "#<number>" format
    - Description truncated to 500 chars
    - Assignee from first element of assignees array, or null
    - Empty input array returns { issues: {}, syncedAt }
    - All output issues have tracker: "github"

    fetchDelta:
    - fetchDelta(projectConfig, lastSyncedAt) calls gh api repos/{repoKey}/issues with since, state=all, per_page=100
    - Uses execFileSync with no shell (array args) matching existing fetchAll pattern
    - Returns normalizeGitHubApiIssues result
    - 15 second timeout matching existing adapter

    normalizeLinearDelta (linear-adapter.js):
    - normalizeLinearDelta(linearIssues, teamKey, lastSyncedAt) filters issues to only those where updatedAt > lastSyncedAt
    - Passes filtered issues to existing normalizeLinearIssues for normalization
    - Returns { issues, syncedAt } same shape as normalizeLinearIssues
    - Issues with updatedAt <= lastSyncedAt are excluded
    - Issues with missing/null updatedAt are excluded
    - Empty result when all issues are older than lastSyncedAt
  </behavior>
  <implementation>
    GitHub adapter (github-adapter.js):

    1. Add normalizeGitHubApiIssues(rawIssues) function -- SEPARATE from existing normalizeGitHubIssues. This handles the snake_case fields from gh api. For each raw item: skip if raw.pull_request exists, key is "#" + raw.number, map raw.state lowercase ("open" -> "started", "closed" -> "completed"), use raw.updated_at (snake_case) for updatedAt field, reuse extractPriorityFromLabels(raw.labels || []), assignee from raw.assignees?.[0]?.login || null, description from (raw.body || "").slice(0, 500), tracker: "github". Return { issues, syncedAt: new Date().toISOString() }.

    2. Add fetchDelta(projectConfig, lastSyncedAt) function -- execFileSync('gh', ['api', `repos/${projectConfig.repoKey}/issues`, '--method', 'GET', '-f', `since=${lastSyncedAt}`, '-f', 'state=all', '-f', 'per_page=100'], { encoding: 'utf8', timeout: 15000, stdio: ['pipe', 'pipe', 'pipe'] }). Parse JSON output, pass to normalizeGitHubApiIssues. Does NOT wrap in try/catch (caller handles errors, same pattern as fetchAll).

    3. Add both to module.exports.

    Linear adapter (linear-adapter.js):

    1. Add normalizeLinearDelta(linearIssues, teamKey, lastSyncedAt) function. Filter linearIssues to those where new Date(raw.updatedAt).getTime() > new Date(lastSyncedAt).getTime(). Skip items with falsy/missing updatedAt. Pass filtered array to existing normalizeLinearIssues(filtered, teamKey). Return the result.

    2. Add to module.exports.
  </implementation>
</feature>
