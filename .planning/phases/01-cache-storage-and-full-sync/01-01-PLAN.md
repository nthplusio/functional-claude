4. Create `cache-store.js` as a CommonJS module (`'use strict'`, `require('fs')`, `require('path')`). Follow existing codebase patterns exactly:
   - Use the polyglot shebang pattern from pm-session-start.js? NO -- this is a library module, not an executable hook. No shebang needed.

5. `getCacheDir(slug, cacheRoot?)`:
   - Returns `path.join(root, slug)`
   - Creates directory with `fs.mkdirSync(dir, { recursive: true })` wrapped in try/catch
   - Returns the path even if mkdir fails (caller handles)

6. `writeIssues(slug, data, cacheRoot?)`:
   - Get filePath from `path.join(getCacheDir(slug, root), 'issues.json')`
   - Temp path: `filePath + '.tmp.' + process.pid`
   - Bak path: `filePath + '.bak'`
   - Serialize: `JSON.stringify(data, null, 2)`
   - Write to temp: `fs.writeFileSync(tmpPath, json, 'utf8')`
   - Backup current (best-effort): if filePath exists, `fs.copyFileSync(filePath, bakPath)` in try/catch
   - Atomic rename: `fs.renameSync(tmpPath, filePath)`
   - Clean up temp on failure: try `fs.unlinkSync(tmpPath)` in catch
   - Return `true` on success, `false` on failure
   - Entire function body wrapped in outer try/catch returning `false`

7. `readIssues(slug, cacheRoot?)`:
   - Get filePath from `path.join(getCacheDir(slug, root), 'issues.json')`
   - Try: read + parse primary file, validate it has `version` and `issues` fields, return data
   - Catch: try .bak file with same validation
   - Catch: return `null`
   - Never throws

8. Run tests -- they MUST all pass (GREEN). Commit: `feat(01-01): implement cache-store with atomic writes and fail-open`
