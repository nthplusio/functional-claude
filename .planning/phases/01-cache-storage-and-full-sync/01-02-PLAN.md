3. Create github-adapter.js as CommonJS module. Two exported functions:

fetchAll(projectConfig):
- Extract repoKey from projectConfig (e.g., 'nthplusio/functional-claude')
- Use the safe execFileSync pattern from pm-session-start.js (no shell, array args). Call gh with args: ['issue', 'list', '--repo', repoKey, '--json', 'number,title,state,updatedAt,assignees,labels,body', '--state', 'all', '--limit', '200']. Set encoding:'utf8' and timeout:15000.
- Parse JSON output, pass to normalizeGitHubIssues
- Add version: CACHE_VERSION and tracker: 'github' to the result (ready for writeIssues)
- Do NOT wrap in try/catch -- let caller handle errors (fail-open is the sync orchestrator's job, not the adapter's)
- Import CACHE_VERSION from cache-store.js

normalizeGitHubIssues(rawIssues):
- Iterate raw issues from gh issue list --json output
- For each issue build a NormalizedIssue:
  - id: '#' + raw.number
  - title: raw.title
  - status: raw.state.toLowerCase() === 'open' ? 'started' : 'completed' (lowercase FIRST to handle both OPEN and open)
  - priority: Extract from labels array -- look for labels with name matching priority:urgent (1), priority:high (2), priority:medium (3), priority:low (4). Default 0.
  - assignee: raw.assignees?.[0]?.login || null
  - description: (raw.body || '').slice(0, 500)
  - updatedAt: raw.updatedAt
  - tracker: 'github'
- Key issues by id in an object (not array)
- Return { issues, syncedAt: new Date().toISOString() }

Also export a helper extractPriorityFromLabels(labels) for testability.
